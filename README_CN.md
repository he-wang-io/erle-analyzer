# ERLE 离线分析工具

**Echo Return Loss Enhancement Offline Analyzer**

专业的 AEC（声学回声消除）性能评估工具，用于离线测试和分析 ERLE 指标。

---

## 功能特点

### 核心功能

| 功能        | 说明                                      |
| --------- | --------------------------------------- |
| **双格式支持** | 支持 WAV 和 PCM 格式输入，自动识别                  |
| **分段计算**  | 默认 100ms 分段，可配置                         |
| **能量门限**  | -46 dBFS 阈值（符合 ITU-T G.168 和 WebRTC 标准） |
| **静默检测**  | 参考信号和麦克风信号双重检测，排除无效段                    |
| **双讲检测**  | ERLE < -3 dB 时自动排除双讲段                   |
| **统计分析**  | 平均值、标准差、最小值、最大值、中位数、P95、P99             |
| **评级系统**  | 优秀/良好/一般/较差 四级评估                        |
| **直方图**   | 可视化 ERLE 分布                             |
| **批量分析**  | 支持目录遍历和汇总报告                             |

### 输出格式

- **文本报告**：完整的分析报告（含直方图）
- **JSON 报告**：结构化数据，便于后续处理
- **CSV 报告**：批量分析结果表格
- **Markdown 报告**：美观的汇总报告

## 安装说明

### 系统要求

- Python 3.7+
- 无需额外依赖（仅使用 Python 标准库）

### 安装步骤

```bash
# 1. 克隆或下载项目
cd /path/to/erle-analyzer

# 2. 确保脚本可执行（可选）
chmod +x erle_analyzer.py
chmod +x erle_batch_analyze.py
```

## 快速开始

### 单文件分析

#### 基本用法（仅需 Rec 和 Out）

```bash
# 最简单：只用 rec 和 out 文件
python erle_analyzer.py -r rec.wav -o out.wav

# 保存报告
python erle_analyzer.py -r rec.wav -o out.wav \
    --report result.txt
```

#### 带参考信号（更准确的静默检测）

```bash
# 有 ref 文件时，可以更准确地排除远端静默段
python erle_analyzer.py -r rec.wav -f ref.wav -o out.wav \
    --report result.txt --json result.json
```

#### WAV 格式（推荐）

```bash
# WAV 格式自动识别采样率
python erle_analyzer.py -r rec.wav -o out.wav
```

#### PCM 格式

```bash
# 指定采样率（8kHz）
python erle_analyzer.py -r rec.pcm -o out.pcm \
    --sample-rate 8000

# 指定采样率（16kHz）
python erle_analyzer.py -r rec.pcm -o out.pcm \
    --sample-rate 16000
```

### 批量分析

```bash
# 分析目录中的所有测试文件
python erle_batch_analyze.py /path/to/test_audio

# 保存多种格式报告
python erle_batch_analyze.py /path/to/test_audio \
    --csv results.csv \
    --md results.md
```

## 技术原理

### ERLE 计算公式

```
ERLE = 10 × log10(P_rec / P_out)  [dB]

其中:
- P_rec = 麦克风输入信号功率（包含回声）
- P_out = AEC 处理后输出信号功率（残余回声）
```

### 能量阈值（符合 ITU-T G.168 和 WebRTC 标准）

| 参数          | 阈值       | 说明                        |
| ----------- | -------- | ------------------------- |
| **参考信号阈值**  | -46 dBFS | 低于此值判定为静默段（仅提供 ref 文件时检测） |
| **麦克风信号阈值** | -46 dBFS | 低于此值判定为静默段                |
| **双讲检测阈值**  | -3 dB    | ERLE 低于此值判定为双讲段           |

### 分段计算流程

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1: 读取音频文件（WAV 或 PCM）                            │
│  - Rec（必需）：麦克风输入信号                               │
│  - Out（可选）：AEC 输出信号（未提供时使用 Rec 代替）           │
│  - Ref（可选）：参考信号（用于远端静默检测）                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 2: 按 100ms 分段（默认）                                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3: 对每段进行能量检查（如果有 Ref 文件）                   │
│  - 参考信号 < -46 dBFS ? → 排除（静默）                       │
│  - 麦克风信号 < -46 dBFS ? → 排除（静默）                      │
│                                                               │
│  如果没有 Ref 文件，只检查麦克风信号                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 4: 计算有效段的 ERLE                                     │
│  - ERLE = 10 × log10(P_rec / P_out)                         │
│  - ERLE < -3 dB ? → 排除（双讲）                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 5: 统计分析                                              │
│  - 平均值、标准差、最小值、最大值                              │
│  - 中位数、P95、P99 百分位数                                  │
│  - 直方图分布                                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 6: 生成报告                                              │
│  - 文本报告（含直方图）                                     │
│  - JSON 报告（结构化数据）                                    │
│  - CSV 报告（批量分析）                                      │
└─────────────────────────────────────────────────────────────┘
```

### 为什么排除静默段和双讲段？

**静默段问题**：

```
双端静默时:
- Ref ≈ 0（远端静音）
- Rec ≈ 本地噪声
- Out ≈ Rec（AEC 无回声可消除）

ERLE = 10 × log10(P_rec / P_out) ≈ 10 × log10(1) = 0 dB

→ 静默段会严重拉低整体 ERLE 评估
```

**双讲段问题**：

```
双端同时讲话时:
- Rec = 近端语音 + 回声
- Out = 近端语音 + 残余回声

由于近端语音加入，可能导致 Out > Rec
ERLE = 10 × log10(P_rec / P_out) < 0 dB

→ 双讲段会产生负的 ERLE 值，干扰评估
```
